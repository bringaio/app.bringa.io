# Quality Loop Workflow

Use this whenever work could move the repository closer to or farther from the hyperoptimum.

1. Define the smallest coherent outcome for the current task.
2. Identify the source of truth for every durable fact touched.
3. Check whether docs or `.agents/` need to change with the code.
4. Implement only the scoped improvement.
5. Verify with commands that actually prove the claim.
6. Commit with a Conventional Commit message.
7. Push immediately unless the user explicitly asked for local-only work or the remote is unavailable.
8. Put useful follow-up ideas in `docs/optimization-options.md`, including uncertainty, side effects, and research needs when relevant.

Stop and ask before expanding scope into unrelated refactors, production data access, legal rewrites, or irreversible operations.
