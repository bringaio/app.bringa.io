# Roadmap

Bringa starts as a focused sharing and borrowing app for communities. The long-term direction is a federated, decentralized, agent-friendly system for sharing and borrowing anything communities can responsibly manage.

## Foundation

- Public JSONC configuration for branding, links, legal defaults, media limits, Supabase public values, and feature switches.
- Clear fork/upstream workflow for deployments such as CONTEKT.
- Agent rules, skills, workflows, maintenance tasks, and optimization register.
- Reliable local verification and backup preparation.

## Product

- Better dashboard defaults, search actions, breadcrumbs, responsive polish, PWA manifest/icons, and offline-safe fonts.
- Multi-image item handling with previews, title image, newest-first ordering, moderation, and export/download.
- Ownership and visibility policies that survive user deletion without losing operator-owned community contributions.
- Admin views for user items, proposed changes, flags, item version history/restore, muted Telegram notifications, and remaining moderation follow-through.

## Data And Trust

- RLS-first Supabase architecture with RPC for sensitive actions.
- Explicit contribution terms: public-domain intent where possible, fallback operator license where required.
- User export for profile data, item data, borrowing history, and images where policy permits.
- Observability for failures, moderation queues, and notification health without exposing personal data.
