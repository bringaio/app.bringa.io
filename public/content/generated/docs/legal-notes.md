# Legal Notes

This is not legal advice. Each deployment operator should review terms, privacy text, and imprint/contact requirements for its jurisdiction.

## General Direction

For item entries and uploaded images, the intended product model is:

- Users understand contributions as a private gift to the portal/operator.
- Images and item descriptions are contributed with public-domain intent where legally possible.
- Where public-domain dedication is not fully possible, users grant the operator a broad, irrevocable, royalty-free fallback license for operating, documenting, preserving, and improving the portal.
- Personal data remains subject to privacy law and user rights; contribution intent does not remove privacy obligations.
- The upstream default should stay limited to necessary authentication session storage. Avoid a cookie banner unless a deployment adds non-essential analytics, marketing, embedded third-party tracking, or similar optional storage.

## Fork Responsibilities

Forks should configure:

- Operator legal name
- Contact and privacy email
- Jurisdiction
- Terms path
- Repository and issue links
- Whether the deployment is private, public, association-only, or invite-only

## Open Questions

- Does a specific deployment need English-only, German-only, or bilingual terms?
- Should the upstream default be CC0-style language, a custom public-domain intent, or links to established licenses?
- Which user export/delete obligations apply to each deployment?
- Which optional providers, if any, would require consent controls in a specific deployment?
